### 2. Configure Backend Environment Variables

Required variables to set in Render dashboard:

```env
DATABASE_URL=postgresql://postgres.[PROJECT-REF]:[PASSWORD]@aws-1-ap-southeast-2.pooler.supabase.com:6543/postgres?pgbouncer=true
GROQ_API_KEY=your_groq_api_key_here
SUPABASE_URL=https://[PROJECT-REF].supabase.co
SUPABASE_ANON_KEY=your_supabase_anon_key
SUPABASE_SERVICE_ROLE_KEY=your_supabase_service_role_key
```

**Note**: Use your actual values from the `.env` file (not committed to git).

Auto-generated by Render:
- `JWT_SECRET` (will be auto-generated)
- `PORT` (automatically set to 10000)

### 3. Configure Frontend Environment Variables

```env
NEXT_PUBLIC_SUPABASE_URL=https://[PROJECT-REF].supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key
```

**Note**: Use your actual values from the `.env.local` file (not committed to git).

Note: `NEXT_PUBLIC_API_URL` and `NEXT_PUBLIC_WS_URL` will be auto-linked from backend.

### 4. Deploy
- [ ] Click "Apply" to start deployment
- [ ] Monitor build logs for both services

### 5. Post-Deployment Configuration

#### Update CORS Settings
Once deployed, get your frontend URL (e.g., `https://ligma-frontend.onrender.com`) and add to backend environment variables:

```env
ALLOWED_ORIGINS=https://ligma-frontend.onrender.com
FRONTEND_URL=https://ligma-frontend.onrender.com
PUBLIC_APP_URL=https://ligma-frontend.onrender.com
```

#### Run Database Migrations (if needed)
If this is first deployment:
1. Go to backend service → Shell tab
2. Run:
   ```bash
   cd backend
   npx prisma migrate deploy --schema=src/db/schema.prisma
   ```

Or run SQL scripts directly in Supabase dashboard.

## ⚠️ Important Notes

### Free Tier Limitations
- Services spin down after 15 minutes of inactivity
- First request after spin-down takes 30-60 seconds
- 750 hours/month per service
- Consider upgrading for production use

### WebSocket Support
- Render supports WebSocket on all plans
- Use `wss://` (not `ws://`) in production
- Both `/ws` and `/yjs` endpoints are available

### Database
- Using Supabase PostgreSQL (already configured)
- Connection pooling enabled (pgbouncer=true)
- Ensure Supabase allows connections from Render IPs

### Security
- All secrets stored as environment variables
- JWT_SECRET auto-generated by Render
- CORS properly configured
- Rate limiting enabled
- SSL/TLS enabled by default

## 🆘 Troubleshooting

### Build Fails
- Check build logs in Render dashboard
- Verify all dependencies in package.json
- Ensure Prisma schema path is correct

### Connection Issues
- Verify DATABASE_URL is correct
- Check Supabase connection settings
- Ensure environment variables are set

### WebSocket Not Working
- Verify using `wss://` (not `ws://`)
- Check CORS settings
- Review browser console for errors
