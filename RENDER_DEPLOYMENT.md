# Render Deployment Guide for LIGMA Collaborative CanvaPad

This guide will help you deploy the LIGMA Collaborative CanvaPad application to Render.

## Prerequisites

1. A [Render account](https://render.com) (free tier available)
2. Your GitHub repository connected to Render
3. Supabase database already set up (as per SUPABASE_SETUP.md)

## Deployment Architecture

The application consists of two services:
- **Backend**: Node.js/Express API with WebSocket support (Yjs + custom WS)
- **Frontend**: Next.js application

Both services are defined in `render.yaml` for easy deployment.

## Step 1: Prepare Your Repository

1. Ensure all changes are committed and pushed to GitHub:
   ```bash
   git add .
   git commit -m "Add Render deployment configuration"
   git push origin main
   ```

## Step 2: Create a New Blueprint on Render

1. Go to [Render Dashboard](https://dashboard.render.com/)
2. Click **"New +"** → **"Blueprint"**
3. Connect your GitHub repository
4. Render will automatically detect the `render.yaml` file

## Step 3: Configure Environment Variables

### Backend Service Environment Variables

You'll need to set these in the Render dashboard for the `ligma-backend` service:

**Required:**
- `DATABASE_URL`: Your Supabase PostgreSQL connection string
  ```
  postgresql://postgres.[PROJECT-REF]:[PASSWORD]@aws-1-ap-southeast-2.pooler.supabase.com:6543/postgres?pgbouncer=true
  ```
- `GROQ_API_KEY`: Your Groq API key for AI features
- `SUPABASE_URL`: Your Supabase project URL
  ```
  https://[PROJECT-REF].supabase.co
  ```
- `SUPABASE_ANON_KEY`: Your Supabase anonymous key
- `SUPABASE_SERVICE_ROLE_KEY`: Your Supabase service role key

**Auto-generated:**
- `JWT_SECRET`: Will be auto-generated by Render
- `PORT`: Automatically set to 10000 by Render

**Pre-configured:**
- `NODE_ENV`: production
- `JWT_EXPIRES_IN`: 7d
- `RATE_LIMIT_WINDOW_MS`: 900000
- `RATE_LIMIT_MAX_REQUESTS`: 100
- `LOG_LEVEL`: info

### Frontend Service Environment Variables

The frontend will automatically receive:
- `NEXT_PUBLIC_API_URL`: Auto-linked from backend service
- `NEXT_PUBLIC_WS_URL`: Auto-linked from backend service

You need to manually set:
- `NEXT_PUBLIC_SUPABASE_URL`: Your Supabase project URL
- `NEXT_PUBLIC_SUPABASE_ANON_KEY`: Your Supabase anonymous key

## Step 4: Deploy

1. After configuring environment variables, click **"Apply"**
2. Render will:
   - Create both services
   - Install dependencies
   - Build the applications
   - Deploy them

3. Monitor the deployment logs for any errors

## Step 5: Update CORS Settings

Once deployed, you need to update the backend's CORS configuration:

1. Get your frontend URL from Render (e.g., `https://ligma-frontend.onrender.com`)
2. In the backend service environment variables, add:
   ```
   ALLOWED_ORIGINS=https://ligma-frontend.onrender.com
   FRONTEND_URL=https://ligma-frontend.onrender.com
   PUBLIC_APP_URL=https://ligma-frontend.onrender.com
   ```

## Step 6: Database Setup

If this is your first deployment, you may need to run database migrations:

1. Go to the backend service in Render
2. Open the **Shell** tab
3. Run:
   ```bash
   cd backend
   npx prisma migrate deploy --schema=src/db/schema.prisma
   ```

Alternatively, you can run the SQL scripts directly in Supabase:
- `backend/src/db/ligma_schema.sql`
- `backend/src/db/sharing_schema.sql`
- `backend/src/db/add_canvas_node_table.sql`

## Important Notes

### WebSocket Support
- Render supports WebSocket connections on all plans
- The backend serves both HTTP and WebSocket on the same port
- WebSocket endpoints:
  - `/ws` - Raw WebSocket for presence and events
  - `/yjs` - Yjs WebSocket for CRDT synchronization

### Free Tier Limitations
- Services spin down after 15 minutes of inactivity
- First request after spin-down may take 30-60 seconds
- 750 hours/month of runtime per service
- Consider upgrading to paid plan for production use

### Health Checks
- Backend includes a health check endpoint at `/api/health`
- Render will automatically monitor this endpoint

### Persistent Storage
- Y.js documents are stored in the database (Supabase)
- Local file storage (`backend/data/ydocs/`) is ephemeral on Render
- Ensure all critical data is persisted to the database

## Troubleshooting

### Build Failures
- Check that all dependencies are in `package.json`
- Verify Prisma schema path is correct
- Review build logs in Render dashboard

### Connection Issues
- Verify DATABASE_URL is correct and accessible
- Check that Supabase allows connections from Render IPs
- Ensure WebSocket URLs use `wss://` (not `ws://`) in production

### Environment Variables
- Double-check all required environment variables are set
- Ensure no trailing spaces in values
- Verify Supabase keys are correct
